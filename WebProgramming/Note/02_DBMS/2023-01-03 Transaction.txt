트랜잭션
 - Transaction은 데이터 처리의 한 단위
 - 오라클에서 발생하는 여러 개의 SQL 명령문들을 하나의 논리적인 작업 단위 처리 단위
 하나의 트랜잭션은 All-OR-Nothing 방식으로 처리.
 명령어 중에서 하나의 명령어라도 잘못되었다면 전체를 취소
 트랜잭션이란 개념을 도입한 이유: 데이터의 일관성을 유지하면서 안정적으로 데이터를 복구시키기 위해

트랜잭션 제어를 위한 명령어
COMMIT:  -모든 작업들을 정상적으로 처리하겠다고 확정하는 명령어, 
	 -COMMIT 명령어를 수행하게 되면 하나의 트랜잭션 과정을 종료
ROLLBACK: 작업 중 문제가 발생되어서 트랜잭션의 처리 과정에서 발생한 변경사항을 취소하는 명령어
	  -ROLLBACK 명령어 역시 트랜잭션 과정을 종료
	  -트랜잭션으로 인한 하나의 묶음 처리가 시작되기 이전의 상태로 되돌림
	  -트랜잭션은 여러 개의 물리적인 작업(DML 명령어)들이 모여서
	    이루어지는데 이러한 과정에서 하나의 물리적인 작업이라도 문제가
  	    발생하게 되면 모든 작업을 취소해야 하므로 이들을 하나의
	    논리적인 작업 단위(트랜잭션)로 구성
SAVEPOINT: 현재의 트랜잭션을 작게 분할
	    저장된 SAVEPOINT는 ROLLBACK TO SAVEPOINT 문을 사용하여 표시한 곳까지 ROLLBACK할 수 있음.
	    여러 개의 SQL 문의 실행을 수반하는 트랜잭션의 경우, 트랜잭션 중간 단계에서 세이브포인트를 지정할 수 있음.
